# algorithm
复杂度包括**时间复杂度**和**空间复杂度**
## 时间复杂度
- 常数时间操作：一个操作如果和样本数据量没有关系，每次都是固定时间内完成的操作，叫做常数操作。
    - O(1)也是表示一个特殊复杂度，即任务与算例个数 n 无关。
    - 如果一个操作的执行时间不以具体样本量为转移，每次执行时间都是固定时间。称这样的操作为常数时间的操作。
    - 常见的常数时间操作：
        - 常见的算术运算:+、-、*、/、% 等。
        - 常见的位运算: >>(带符号右移)、>>>(不带符号右移)、<<、|、&、^等。([【极客时间】二进制：不了解计算机的源头，你学什么编程]())
        - 赋值、比较、自增、自减操作等。
        - 数组寻址操作。
- 时间复杂度为一个算法流程中，常数操作数量的一个指标。常用O(读做 big O)来表示。
- 具体来说，先要对一个算法流程非常熟悉，然后去写出这个算法流程中，发生了多少常数操作，进而总结出常数操作数量的表达式。
- 在表达式中，**只要高阶项，不要低阶项，也不要高阶项的系数**，剩下的部分如果为f(N)，那么时间复杂度为O(f(N))。
    - 复杂度相加的时候，选择高者作为结果，也就是O(n^2) + O(n)和O(n^2)表示的是同样的复杂度。
- 为什么只留下最高阶项？
    - 当我们要处理的样本量很大时，会发现低阶项是什么不是最重要的；每一项的系数是什么，不是最重要的；真正重要的就是最高阶项是什么。
    - 这就是时间复杂度的意义，它是衡量算法流程的复杂程度的一种指标，该指标只与数据量有关，与过程之外的优化无关。
- 评价一个算法流程的好坏，先看时间复杂度的指标，然后再分析不同数据样本下的实际运行时间，也就是*常数项时间*
- 时间复杂度与具体的常系数无关，O(n)和O(2n)表示的是同样的复杂度

### 常见时间复杂度
O(1)、O(logN)、O(N)、O(NlogN)、O(N^2)、O(N^3)、...O(N^k)、O(2^N)、O(N!)

**时间复杂度**与代码的结构设计高度相关。  
**空间复杂度**与代码中数据结构的选择高度相关。  

#### 降低复杂度的三个步骤
- 暴力解法：在没有任何时间、空间约束下，完成代码任务的开发。
- 无效操作处理：将代码中的无效计算、无效存储剔除，降低时间或空间复杂度。
    - 递归、二分法、排序算法、动态规划等常用算法思维。
- 时空转换：设计合理数据结构，完成时间复杂度向空间复杂度的转移。
    - 对数据的操作进行细分，常见数据结构的基础知识。
